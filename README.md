# JJXS_Studio — 内容生产系统

一套基于 Proma Agent 的内容生产工作流配置。帮助你按照标准化流程完成数码科技内容的选题、研究、脚本、分镜和复盘。

## 准备

1. 安装 [Proma 桌面应用](https://erlichliu.github.io/Proma)
2. 在 Proma 设置中配置好模型通道
3. 安装 [Obsidian](https://obsidian.md/)，用于管理内容文件

## 部署

**第一步：安装系统规则**

打开 Proma，新建 Agent 会话，发送：

```
请把团队的内容生产系统配置到我的工作区。

1. 备份当前配置（如果有）
2. 从 GitHub 克隆系统仓库：https://github.com/Raymond-Leung7/content-system.git
3. 验证 CLAUDE.md 已加载成功
```

Proma 会自动完成所有操作。之后每次打开新会话，工作流规则自动生效。

**第二步：搭建内容目录**

在 Obsidian 中创建自己的本地仓库（放哪里、叫什么都可以），然后告诉 Proma：

```
这是我的 Obsidian 目录：<你的完整路径>，请帮我建好需要的文件夹
```

**第三步：验证**

问 Proma："我现在是什么角色？" 如果回答包含"东星斑"和"B站数码科技内容"，说明部署成功。

## 目录结构

部署完成后，你的 Obsidian 目录会是这样：

| 目录 | 用途 |
|------|------|
| `00-Inbox/` | 资讯输入 |
| `01-选题/` | 选题池 |
| `02-产品资料/` | 产品研究 + 竞品分析 |
| `04-脚本/` | 脚本文件 |
| `05-分镜表/` | 分镜表 xlsx |
| `06-复盘/` | 复盘报告 |
| `99-Archive/` | 归档 |

## 日常使用

| 你要做什么 | 对 Proma 说 |
|-----------|------------|
| 获取最新规则 | `帮我拉取最新的系统配置` |
| 推配置到仓库（维护者用） | `请把我的本地配置推送到内容系统仓库` |
| 查看仓库变更 | `帮我看看内容系统仓库有什么变更` |

## 仓库文件

| 文件 | 说明 |
|------|------|
| `CLAUDE.md` | 工作流规则、内容标准、引用纪律 — 系统核心 |
| `TEAM.md` | 团队使用详细说明 |
| `.context/team-note.md` | 竞品库、技能清单、踩坑记录 |
| `templates/` | 分镜表模板等 |
| `.gitignore` | 隐私保护：个人记忆、API key、本地配置不会被推送 |

## 隐私说明

你放进 Obsidian 的文件完全由你自己管理，不会共享给任何人。推送到仓库时仅包含规则和模板，不会误提交你的个人记忆或 API key。
