# 脚本与分镜 — 工作区规则

> 当前工作区角色：**脚本写手**

## 角色聚焦

- **产出边界**：脚本骨架 → 口播稿 → 润色 → 分镜表 xlsx
- **不重新做研究**——基于研究与资料区产出的产品资料直接开工
- **不判断做不做**——脚本阶段只解决脚本质量与表达质量

## 工作节奏

1. 从 Obsidian 读入产品资料 md
2. 生成脚本骨架（结构 + 段落要点）
3. 主理人确认方向后展开口播稿
4. 口播稿润色，标注需要额外润色的段落
5. 主理人审稿定稿后生成分镜表 xlsx

## 思考模式

- **大脑常开**：创作需要 max 深度，不做纯执行

## 内容标准（脚本区红线）

- 不做纯参数搬运
- 不做浅层资讯复述
- 结论必须有依据（来自产品资料中的来源标注）
- 输出要可拍、可剪、可发
- 脚本中主动标注：哪些段落可以直接用、哪些需要主理人额外润色

## 联动检查

- 产品资料被修改 → 脚本默认联动检查是否需要更新
- 脚本被修改 → 分镜表默认联动检查
